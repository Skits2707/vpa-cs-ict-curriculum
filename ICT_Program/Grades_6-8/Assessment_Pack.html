Grades 6–8 Assessment Pack

Program: Digital Mastery | Assessment Types: Formative, Summative, Portfolio

Assessment Philosophy

Middle school assessment bridges elementary portfolios and IGCSE examinations. Students receive rubric-based feedback for every project, complete timed practical tasks, and build a comprehensive digital portfolio. Coding assessments use auto-grading platforms (Replit, CodeHS) for immediate feedback with teacher verification.

Coding Assessment Rubric (Auto-Graded + Teacher)

Auto-Graded (by platform):

  • ☐ Code compiles/runs without errors
  • ☐ Passes provided test cases
  • ☐ Runs within time limits

Teacher-Graded:

  • ☐ Variable names are meaningful (not x, y, z)
  • ☐ Code includes comments explaining logic
  • ☐ Solution is efficient (not overly complex)
  • ☐ Handles edge cases (empty input, zero, negatives)

Portfolio Requirements (End of Grade 8)

Students submit a digital portfolio containing:

  • ☐ Best word-processed document (with styles and TOC)
  • ☐ Best spreadsheet (with formulas, charts, validation)
  • ☐ Best presentation (10+ slides, embedded media)
  • ☐ Python project with source code and README
  • ☐ Website (HTML/CSS files or hosted link)
  • ☐ Database file with documentation
  • ☐ Robotics project video or photos
  • ☐ Cybersecurity guide or infographic
  • ☐ Edited video (1+ minute)
  • ☐ Self-reflection essay (200 words): "My ICT Journey"

EAL Support in Middle School Assessment

  • Bilingual technical glossaries: Provided for all units with English/中文/Pinyin
  • Scaffolded rubrics: Bronze/Silver/Gold descriptions include visual exemplars
  • Oral defense option: Students may explain code/projects orally if writing is a barrier
  • Extended time: +15 minutes for practical assessments
  • Peer translation: Bilingual students may assist with instruction (not assessment)
  • Code comments in L1: Students may write comments in Mandarin to demonstrate understanding
