Topic 5.1: Stacks, Queues, Linked Lists, Trees, Graphs 栈、队列、链表、树、图

Week 1 — Linked Lists 链表 Liànbiǎo

LessonContentActivitiesDifferentiation
1Linked list structure: nodes, data field, pointer/reference field, head, tail, null (节点, 指针, 头节点)Draw linked list diagrams; trace traversalBronze: Label diagram; Silver: Trace operations; Gold: Compare to arrays
2Singly linked list operations: insert at start, insert at end, delete, traverse, searchWrite pseudocode for each operation; trace on diagramsBronze: Trace given code; Silver: Write pseudocode; Gold: Handle edge cases (empty list, single node)
3Doubly linked lists: forward and backward traversal; advantages over singly linkedModify singly linked pseudocode for doubly linkedAll: Understand concept; Gold: Implement with error checking
4Circular linked lists; sentinel nodes; comparison of all variantsComparison table: memory, access time, use casesAll: Complete table; Gold: Analyse cache performance
5TOK: A linked list is an abstraction with no physical existence. In what sense can we say it "exists"? Is it a mathematical object, a computational one, or both?Discussion; written reflectionAll: Contribute; Gold: Reference Platonism vs formalism
TOK Knowledge Question: A linked list is defined entirely by relationships (pointers) between nodes, not by the nodes themselves. Does this mean the structure has an existence independent of its contents? Can the form of a data structure be said to exist in a different way from its matter?

Week 2 — Stacks and Queues (Dynamic Implementation) 栈与队列 Zhàn yǔ Duìliè

LessonContentActivities
1Stack as linked list: push (insert at head), pop (remove from head), dynamic sizing (栈的动态实现)Implement dynamic stack; compare to array-based
2Queue as linked list: enqueue (insert at tail), dequeue (remove from head)Implement dynamic queue; trace pointer updates
3Circular queue implementation using arrays; full/empty detectionDesign circular buffer for streaming data
4Priority queue: concept, insertion by priority, dequeuing highest priorityImplement simple priority queue; discuss heap optimisation
5Formative: Implement stack and queue operations in pseudocode (30 marks)Timed; peer mark with criteria

Week 3 — Trees 树 Shù

LessonContentActivities

2Binary trees: each node has at most two children; binary search tree (BST) propertyBuild BST from sequence; trace search operations
3BST operations: insert, search, delete (leaf, one child, two children cases)Write pseudocode; trace complex deletions
4Tree balancing: why it matters; AVL and Red-Black trees (conceptual)Analyse unbalanced vs balanced tree search times
5Formative: BST construction and search trace table (25 marks)Individual; markscheme review

Topic 5.2: Tree Traversal and Graph Algorithms 树遍历与图算法

Week 4 — Tree Traversal 树遍历 Shù Biànlì

LessonContentActivities
1Preorder traversal: root, left, right (前序遍历)Traverse given trees; write traversal sequence
2Inorder traversal: left, root, right; BST inorder yields sorted sequence (中序遍历)Verify: inorder of BST is sorted; prove to peer
3Postorder traversal: left, right, root; deletion order (后序遍历)Determine postorder; relate to expression trees
4Expression trees: constructing from infix; evaluating with postorderBuild tree from expression; evaluate
5TOK: Tree traversals produce different "readings" of the same structure. Is any traversal more "true" than another? What determines the right way to traverse?Discussion; connect to hermeneutics
TOK Knowledge Question: Inorder traversal of a BST produces a sorted list. Is this sorting "discovered" (the order was already there) or "created" (the traversal imposes an order)? What does this suggest about the relationship between representation and knowledge?

Week 5 — Graphs 图 Tú

LessonContentActivities
1Graph terminology: vertex, edge, directed/undirected, weighted, adjacent, degree, path, cycle (顶点, 边, 有向图, 无向图)Model real networks as graphs; identify properties
2Graph representations: adjacency matrix, adjacency list; comparisonRepresent same graph both ways; compare space/time
3Graph applications: social networks, navigation, dependency graphs, state machinesResearch: how does Google Maps represent roads?
4Depth-first search (DFS): algorithm, stack-based, recursive; applicationsTrace DFS on graph; implement recursively
5Breadth-first search (BFS): algorithm, queue-based; shortest path in unweighted graphsTrace BFS; compare to DFS for finding paths

5Formative: Graph algorithm trace and comparison (30 marks)Timed; peer review
