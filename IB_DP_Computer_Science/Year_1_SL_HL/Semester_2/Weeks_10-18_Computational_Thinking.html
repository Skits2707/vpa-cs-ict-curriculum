2Operators: arithmetic (+, −, *, /, MOD, DIV), relational, logicalEvaluate expressions; predict outputsBronze: Integer arithmetic; Silver: Mixed types; Gold: Complex boolean

TOK Knowledge Question: Pseudocode occupies a space between natural language and formal programming languages. When we write pseudocode, are we creating a precise set of instructions, or are we merely suggesting an approach? Can pseudocode be "wrong," or is it too informal to have truth values?

Topic 4.3: Algorithms 算法 Suànfǎ

Week 14 — Searching Algorithms 搜索算法 Sōusuǒ Suànfǎ

LessonContentActivities
1Linear search: algorithm, trace, best/average/worst case (线性搜索)Trace on small arrays; implement in pseudocode
2Binary search: precondition, algorithm, trace; comparison to linearPlay "guess the number"; trace binary search tree
3Algorithm efficiency: Big O notation introduction (HL depth)Compare O(n) vs O(log n) graphically
4Trace tables: constructing and using to verify algorithm correctness (追踪表)Complete trace tables for given algorithms
5TOK: Is efficiency a property of an algorithm itself, or only of its implementation? Can we know an algorithm is optimal?Discussion; link to P vs NP
TOK Knowledge Question: We say binary search is "more efficient" than linear search. But efficiency depends on sorted data. Is the claim "binary search is better" an absolute truth, or is it relative to assumptions about the world? How do hidden assumptions shape what we claim to know in computer science?
