Lesson 1 (40 min): Variables, Constants & Data Types 变量、常量与数据类型

Learning Objectives:
• Declare variables with correct data types 用正确的数据类型声明变量
• Distinguish between variables and constants 区分变量和常量
• Select appropriate data types for given data 为给定数据选择合适的数据类型

Starter (5 min): "Do Now" — Given: age=16, price=19.99, name="Alice", isStudent=TRUE. What types are these?

Main Activity (20 min):
• Variable: Named memory location whose value can change. 变量：命名的内存位置，值可以改变。
• Constant: Named value that does not change during execution. 常量：执行期间不改变的命名值。
• CIE data types:
— INTEGER: Whole numbers (-3, 0, 42). 4 bytes typically.
— REAL: Decimal numbers (3.14, -0.5). Floating-point representation.
— STRING: Text ("Hello", "123"). Sequence of characters.
— CHAR: Single character ('A', '7').
— BOOLEAN: TRUE or FALSE. 1 byte.
— DATE: Calendar dates.
• Declaration syntax: DECLARE Age : INTEGER
• Constant syntax: CONSTANT Pi = 3.14159
• Assignment: Age ← 16 (the arrow shows direction of data flow)
• Naming conventions: Meaningful names, no spaces, CamelCase or underscores.
• Type errors: Age ← "Hello" → Type mismatch!

Activity (10 min): Given scenario (student registration), students declare 8 variables with correct types.

Plenary (5 min): "Why use constants instead of hard-coding values?" Discuss: maintainability, readability, single point of change.

Main Activity (20 min):
• Input: INPUT Name — prompts user, stores response in variable.

— TRUE AND FALSE = FALSE

1. Parentheses ()
2. NOT
3. *, /, DIV, MOD, AND
4. +, -, OR
5. Comparison operators
• Example: 3 + 4 * 5 = 23 (not 35). (3 + 4) * 5 = 35.

Week 2: Selection & Iteration 选择与循环

Lesson 1 (40 min): Selection — IF, ELSE, Nested IF, CASE 选择结构：IF、ELSE、嵌套IF、CASE

Learning Objectives:
• Write pseudocode using IF...THEN...ELSE 使用IF...THEN...ELSE编写伪代码
• Use nested IF for multiple conditions 使用嵌套IF处理多个条件
• Apply CASE for multiple discrete options 应用CASE处理多个离散选项

Starter (5 min): "Should I bring an umbrella?" Flowchart decision: Is it raining? Is it cloudy? Will it rain?

Main Activity (20 min):
• IF...THEN: Single condition, single action.
IF Score >= 50 THEN
    OUTPUT "Pass"
ENDIF

• IF...THEN...ELSE: Two paths.
IF Score >= 50 THEN
    OUTPUT "Pass"
ELSE
    OUTPUT "Fail"
ENDIF

• Nested IF: Multiple conditions.
IF Score >= 80 THEN
    OUTPUT "A"
ELSE
    IF Score >= 70 THEN
        OUTPUT "B"
    ELSE
        IF Score >= 60 THEN
            OUTPUT "C"
        ELSE
            OUTPUT "D"
        ENDIF
    ENDIF
ENDIF

• CASE: Cleaner for multiple discrete values.
CASE Grade OF
    "A" : OUTPUT "Excellent"
    "B" : OUTPUT "Good"
    "C" : OUTPUT "Satisfactory"
    "D" : OUTPUT "Pass"
    OTHERWISE OUTPUT "Fail"
ENDCASE

• Comparison: When to use IF vs CASE? CASE for discrete values (menu choices, grades, days). IF for ranges and complex conditions.
• Common error: Missing ENDIF. Every IF needs matching ENDIF. CIE is strict about this.

Activity (10 min): Write pseudocode for: "Input month number (1-12), output season." First with nested IF, then with CASE. Compare readability.

Plenary (5 min): "Why is CASE better than 12 nested IFs for menu selection?" Discuss readability, maintainability, efficiency.

Homework: Write pseudocode for a simple calculator: input two numbers and operator (+, -, *, /), output result. Use CASE for operator selection.

Common Misconceptions:
• ❌ "ELSE is optional" — Technically yes, but omitting it when needed causes logic errors
• ❌ "You can use CASE with ranges like CASE Score OF 0-59" — CIE CASE uses discrete values, not ranges. Use IF for ranges.
Lesson 2 (40 min): Iteration — FOR, WHILE, REPEAT 循环结构：FOR、WHILE、REPEAT

Learning Objectives:
• Write programs using FOR loops 使用FOR循环编写程序
• Write programs using WHILE loops 使用WHILE循环编写程序
• Write programs using REPEAT...UNTIL loops 使用REPEAT...UNTIL循环编写程序
• Select appropriate loop type for given problem 为给定问题选择合适的循环类型

Starter (5 min): "I want to print numbers 1 to 10." Could use 10 OUTPUT statements. Better way? Introduce loops.

Main Activity (20 min):
• FOR loop: Definite iteration (known number of repetitions).
FOR Count = 1 TO 10
    OUTPUT Count
ENDFOR

— Can count down: FOR Count = 10 TO 1 STEP -1
— Can step by other amounts: FOR Count = 0 TO 100 STEP 5
• WHILE loop: Indefinite iteration (condition checked BEFORE each iteration).
DECLARE Password : STRING
Password ← ""
WHILE Password <> "secret" DO
    OUTPUT "Enter password:"
    INPUT Password
ENDWHILE
OUTPUT "Access granted"

— May execute zero times if condition initially false.
• REPEAT...UNTIL loop: Indefinite iteration (condition checked AFTER each iteration).
DECLARE Choice : STRING
REPEAT
    OUTPUT "Menu: A-Add, D-Delete, Q-Quit"
    INPUT Choice
UNTIL Choice = "Q"

— Always executes at least once.
• Comparison table:
LoopWhen to UseCheckMin Executions
FORKnown countAutomatic0 (if start > end)
WHILEUnknown count, may be zeroBefore0
REPEATUnknown count, at least onceAfter1

Activity (10 min): For each scenario, choose loop type and write pseudocode: (a) Print 5 times table, (b) Guess number game, (c) Input valid date.

Plenary (5 min): "What happens if you forget to update the condition in a WHILE loop?" Infinite loop! Demo with WHILE TRUE DO...

Homework: Write 3 programs: (1) FOR loop — sum of first 100 integers, (2) WHILE loop — keep doubling until > 1000, (3) REPEAT loop — menu system.

Bilingual Keywords:
• Loop 循环 (xúnhuán)
• Iteration 迭代 (diédài)
• Condition 条件 (tiáojiàn)
• Infinite loop 无限循环 (wúxiàn xúnhuán)
Lesson 3 (40 min): Nested Loops & Complex Patterns 嵌套循环与复杂模式

Learning Objectives:
• Write nested loop structures 编写嵌套循环结构
• Use loops to process patterns and tables 使用循环处理模式和表格
• Trace nested loops with trace tables 用追踪表追踪嵌套循环

Starter (5 min): How would you print a 5×5 grid of asterisks? One loop for rows, one for columns.

Main Activity (20 min):
• Nested FOR loops: Outer loop controls rows, inner loop controls columns.
FOR Row = 1 TO 5
    FOR Col = 1 TO 5
        OUTPUT "*"  // Without newline
    ENDFOR
    OUTPUT ""     // Newline
ENDFOR

• Triangle patterns:
FOR Row = 1 TO 5
    FOR Col = 1 TO Row
        OUTPUT "*"
    ENDFOR
    OUTPUT ""
ENDFOR

• Times table grid:
FOR i = 1 TO 12
    FOR j = 1 TO 12
        OUTPUT i * j, " "
    ENDFOR
    OUTPUT ""
ENDFOR

• Trace table for nested loops: Track outer loop variable, inner loop variable, output.
• Common error: Forgetting to reset inner loop variables or using same variable name for both loops.

Activity (10 min): Write pseudocode for multiplication table (1-12). Trace for first 3 rows.

Plenary (5 min): "How many times does the inner loop body execute in a 10×10 nested loop?" 100 times. Time complexity O(n²).

Homework: Write pseudocode to print: (a) Hollow square, (b) Number pyramid, (c) Chessboard pattern (alternating X and O).

Bronze: Complete trace table for given nested loop; identify pattern
Silver: Write nested loop for simple pattern (solid rectangle)
Gold: Write nested loop for complex pattern (diamond, Pascal's triangle) with efficient logic

Week 3: Nested Structures, Compound Conditions & Flags 嵌套结构、复合条件与标志

Lesson 1 (40 min): Compound Conditions & Logical Operators 复合条件与逻辑运算符

Learning Objectives:
• Combine conditions using AND, OR, NOT 使用AND、OR、NOT组合条件
• Construct truth tables for complex conditions 为复杂条件构建真值表
• Apply De Morgan's laws 应用德摩根定律

Starter (5 min): "You can go to the party if you finish homework AND clean your room." Is going with just homework done? No. Need BOTH.

Main Activity (20 min):
• Compound conditions: Multiple conditions combined.
• AND: Both must be true. IF Age >= 18 AND HasLicense = TRUE THEN
• OR: At least one must be true. IF Day = "Saturday" OR Day = "Sunday" THEN
• NOT: Reverses condition. IF NOT IsMember THEN (same as IF IsMember = FALSE THEN)
• Truth tables for AND, OR, NOT:
ABA AND BA OR BNOT A
TTTTF
TFFTF
FTFTT
FFFFT

• De Morgan's Laws:
— NOT (A AND B) = (NOT A) OR (NOT B)
— NOT (A OR B) = (NOT A) AND (NOT B)
— Example: NOT (Age > 18 AND HasLicense) = Age <= 18 OR NOT HasLicense
• Short-circuit evaluation: In A AND B, if A is false, B is not evaluated. In A OR B, if A is true, B is not evaluated.

Activity (10 min): Simplify conditions using De Morgan's laws. Evaluate compound conditions.

Plenary (5 min): "Why is short-circuit evaluation useful?" Discuss: efficiency, preventing errors (e.g., x <> 0 AND (10/x > 2)).

Homework: Create truth table for (A AND B) OR (NOT C) with 8 rows. Write pseudocode using compound conditions for login validation (username AND password AND NOT locked).

Common Misconceptions:
• ❌ "A AND B means either A or B is true" — No, BOTH must be true
• ❌ "NOT (A AND B) = NOT A AND NOT B" — Wrong! It's NOT A OR NOT B (De Morgan's)
Lesson 2 (40 min): Flag Variables & State Tracking 标志变量与状态跟踪

Learning Objectives:
• Use Boolean flags to control program flow 使用布尔标志控制程序流程
• Implement search algorithms with flags 使用标志实现搜索算法
• Design state-based solutions 设计基于状态的解决方案

Starter (5 min): "Find the word 'apple' in a list of 100 fruits." How do you know if you found it or not? Introduce flag.

Main Activity (20 min):
• Flag variable: Boolean variable tracking whether something has occurred. 标志变量：跟踪某事是否发生的布尔变量。
• Search example:
DECLARE Found : BOOLEAN
Found ← FALSE
FOR i = 1 TO 100
    IF Fruits[i] = "Apple" THEN
        Found ← TRUE
    ENDIF
ENDFOR
IF Found = TRUE THEN
    OUTPUT "Apple is in the list"
ELSE
    OUTPUT "Apple not found"
ENDIF

• Optimised search (exit early when found):
Found ← FALSE
i ← 1
WHILE i <= 100 AND Found = FALSE DO
    IF Fruits[i] = "Apple" THEN
        Found ← TRUE
    ELSE
        i ← i + 1
    ENDIF
ENDWHILE

• State tracking: Using flags for multi-stage processes.
— Example: Password validation — needs uppercase, lowercase, digit, special char.
— Four flags, all must be true for valid password.

Activity (10 min): Write pseudocode to check if a number is prime using flag. Test with 17 and 18.

Plenary (5 min): "When is a flag better than a direct condition?" Discuss: multiple checks, early exit, complex state tracking.

Homework: Write pseudocode for password validator with 4 flags (upper, lower, digit, length).

Bilingual Keywords:
• Flag 标志 (biāozhì)
• State 状态 (zhuàngtài)
• Search 搜索 (sōusuǒ)
• Valid 有效 (yǒuxiào)
Lesson 3 (40 min): Problem Solving with Pseudocode 用伪代码解决问题

Learning Objectives:
• Apply stepwise refinement to problems 对问题应用逐步求精

DECLARE Price, DiscountRate, TaxRate, Total : REAL

Week 4: Subroutines — Procedures & Functions 子程序：过程与函数

Lesson 1 (40 min): Procedures 过程

Learning Objectives:
• Explain purpose of procedures 解释过程的目的
• Write pseudocode for procedures with parameters 编写带参数的过程伪代码
• Understand scope of variables 理解变量的作用域

Starter (5 min): You need to greet 10 different students. Without copy-pasting code 10 times, how?

Main Activity (20 min):
• Procedure: Named block of code that performs a task. No return value. 过程：执行任务的命名代码块，无返回值。
• Syntax:
PROCEDURE Greet(Name : STRING)
    OUTPUT "Hello, ", Name, "!"
ENDPROCEDURE

// Main program
CALL Greet("Alice")
CALL Greet("Bob")

• Parameters: Values passed to procedure. 参数：传递给过程的值。
• Multiple parameters:
PROCEDURE DisplayResult(Subject : STRING, Score : INTEGER, Grade : STRING)
    OUTPUT Subject, ": ", Score, " (", Grade, ")"
ENDPROCEDURE

• Scope: Variables declared inside procedure are LOCAL (only accessible there). 作用域：过程内声明的变量是局部的。
• Global variables: Declared outside, accessible everywhere. 全局变量：外部声明，到处可访问。
• Best practice: Minimise global variables. Use parameters and local variables. 最佳实践：最小化全局变量。

Activity (10 min): Write procedure to print a rectangle of asterisks given width and height. Call it 3 times with different sizes.

Plenary (5 min): "Why are global variables considered bad practice?" Discuss: unintended side effects, hard to debug, not reusable.

Homework: Write 3 procedures for a calculator: Add, Subtract, Multiply. Each takes 2 parameters and outputs result.

Bilingual Keywords:
• Procedure 过程 (guòchéng)
• Parameter 参数 (cānshù)
• Argument 实参 (shícān)
• Scope 作用域 (zuòyòng yù)
• Local 局部的 (júbù de)
• Global 全局的 (quánjú de)
Lesson 2 (40 min): Functions 函数

Learning Objectives:
• Distinguish functions from procedures 区分函数与过程
• Write functions that return values 编写返回值的函数
• Use functions in expressions 在表达式中使用函数

Starter (5 min): In maths, f(x) = 2x + 1. What is f(3)? Functions in programming are similar — they RETURN a value.

Main Activity (20 min):
• Function: Subroutine that returns a value to the calling code. 函数：向调用代码返回值的子程序。
• Syntax:
FUNCTION Max(a : INTEGER, b : INTEGER) RETURNS INTEGER
    IF a > b THEN
        RETURN a
    ELSE
        RETURN b
    ENDIF
ENDFUNCTION

// Usage
Larger ← Max(5, 3)  // Larger = 5
OUTPUT Max(10, 20)  // Outputs 20

• Key difference from procedure: Function has RETURN statement and RETURNS type. 关键区别：函数有RETURN语句和返回类型。
• Functions can be used in expressions: Result ← Max(x, y) + Max(a, b)
• Common function examples:
— FUNCTION IsEven(n : INTEGER) RETURNS BOOLEAN
— FUNCTION CircleArea(radius : REAL) RETURNS REAL
— FUNCTION GradeFromScore(Score : INTEGER) RETURNS STRING
• Functions should be PURE when possible: Same input always gives same output. No side effects. 纯函数：相同输入总是产生相同输出。

Activity (10 min): Write functions: (a) Fahrenheit to Celsius, (b) IsLeapYear, (c) Factorial (iterative).

Plenary (5 min): "Can a function contain OUTPUT statements?" Discuss: technically yes, but functions should calculate and return, not display. Mixed responsibilities.

Homework: Write function library: AreaOfRectangle, AreaOfTriangle, AreaOfCircle. Main program asks user which shape, calls appropriate function.

Common Misconceptions:
• ❌ "Functions and procedures are the same" — Functions return values; procedures don't (or shouldn't)
• ❌ "You can only RETURN at the end" — RETURN can appear anywhere, but single exit point is cleaner
Lesson 3 (40 min): Parameter Passing & Practical Test 参数传递与实践测试

Learning Objectives:
• Explain pass-by-value and pass-by-reference 解释值传递和引用传递
• Analyse effects of parameter passing on variables 分析参数传递对变量的影响
• Complete practical programming test 完成实践编程测试

Starter (5 min): You give a friend a photocopy of your essay (value) vs the original document (reference). What's the difference if they change it?

Main Activity (20 min):
• Pass by value: Copy of data sent to subroutine. Original unchanged. 值传递：发送数据的副本，原始数据不变。
• Pass by reference: Address sent. Subroutine can modify original. 引用传递：发送地址，子程序可以修改原始数据。
• CIE pseudocode: By default, parameters are pass by value. To indicate pass by reference, some systems use BYREF keyword (context-dependent in CIE).
• Example demonstrating difference:
PROCEDURE Triple(x : INTEGER)
    x ← x * 3
    OUTPUT "Inside: ", x
ENDPROCEDURE

// Main
Num ← 5
CALL Triple(Num)
OUTPUT "Outside: ", Num  // Num is still 5 (pass by value)

• If pass by reference, Num would become 15.
• Implications: Use pass by value for input data. Use pass by reference when subroutine needs to modify caller's data.

Activity (10 min): Predict output for 3 parameter-passing scenarios. Test understanding.

Plenary (5 min): "Why is pass by reference more efficient for large arrays?" Discuss: avoids copying entire array.

Week 5: Recursion 递归 (Dìguī)

Lesson 1 (40 min): Introduction to Recursion 递归入门

Learning Objectives:
• Define recursion and identify recursive patterns 定义递归并识别递归模式
• Explain base case and recursive case 解释基准情况和递归情况
• Trace simple recursive functions 追踪简单递归函数

Starter (5 min): Russian dolls — each doll contains a smaller doll. When does it stop? Smallest doll = base case.

Main Activity (20 min):
• Recursion: Function calls itself to solve smaller instance of same problem. 递归：函数调用自身解决同类更小问题。
• Two essential components:
1. Base case: Condition that stops recursion. 基准情况：停止递归的条件。
2. Recursive case: Function calls itself with simpler input. 递归情况：函数用更简单的输入调用自身。
• Factorial example:
FUNCTION Factorial(n : INTEGER) RETURNS INTEGER
    IF n <= 1 THEN
        RETURN 1
    ELSE
        RETURN n * Factorial(n - 1)
    ENDIF
ENDFUNCTION

Factorial(4) = 4 × Factorial(3) = 4 × 3 × Factorial(2) = 4 × 3 × 2 × Factorial(1) = 4 × 3 × 2 × 1 = 24
• Stack frames: Each recursive call adds frame to call stack. When base case reached, frames unwind. 栈帧：每次递归调用向调用栈添加一帧。
• Memory risk: Deep recursion causes stack overflow. 内存风险：深层递归导致栈溢出。

Activity (10 min): Trace Factorial(5) on paper. Show stack frames at each level.

Plenary (5 min): "What happens if you forget the base case?" Infinite recursion → stack overflow → crash.

Homework: Write recursive function to calculate sum of first n integers. Trace for n=4.

Common Misconceptions:
• ❌ "Recursion is just a loop" — Different mechanism (stack vs iteration), different use cases
• ❌ "Recursion is always better" — Iteration often more efficient (no stack overhead)
Lesson 2 (40 min): Recursive Algorithms — Fibonacci & Search 递归算法：斐波那契与搜索

Learning Objectives:
• Implement Fibonacci recursively 递归实现斐波那契
• Compare recursive and iterative solutions 比较递归和迭代解决方案
• Understand when recursion is appropriate 理解何时使用递归

Starter (5 min): Fibonacci sequence: 1, 1, 2, 3, 5, 8, 13... Each number = sum of two previous. How to code this?

Main Activity (20 min):
• Fibonacci recursive:
FUNCTION Fibonacci(n : INTEGER) RETURNS INTEGER
    IF n <= 2 THEN
        RETURN 1
    ELSE
        RETURN Fibonacci(n - 1) + Fibonacci(n - 2)
    ENDIF
ENDFUNCTION

• Trace Fibonacci(4):
Fib(4) = Fib(3) + Fib(2)
Fib(3) = Fib(2) + Fib(1) = 1 + 1 = 2
Fib(2) = 1
Fib(4) = 2 + 1 = 3
• Problem: Recursive Fibonacci is very inefficient! Fib(5) calls Fib(3) twice. Exponential time O(2^n).
• Iterative Fibonacci: O(n) time, O(1) space. Much better for large n.
• When to use recursion:
— Tree/graph traversal (natural fit)
— Divide and conquer (merge sort, quick sort)
— Problems with recursive structure (file systems, nested menus)
— When code clarity matters more than performance

Lesson 2 (40 min): Linear Search & Bubble Sort 线性搜索与冒泡排序

Learning Objectives:
• Implement linear search algorithm 实现线性搜索算法
• Implement bubble sort algorithm 实现冒泡排序算法
• Analyse efficiency of simple algorithms 分析简单算法的效率

Starter (5 min): Find "Zhang" in unordered class list. How? Check each name one by one. Introduce linear search.

Main Activity (20 min):
• Linear Search:
FUNCTION LinearSearch(arr : ARRAY, n : INTEGER, target : STRING) RETURNS INTEGER
    DECLARE i : INTEGER
    FOR i = 1 TO n
        IF arr[i] = target THEN
            RETURN i
        ENDIF
    ENDFOR
    RETURN -1  // Not found
ENDFUNCTION

— Best case: O(1) — target is first element.
— Worst case: O(n) — target is last or not present.
— Average: O(n).
• Bubble Sort:
PROCEDURE BubbleSort(arr : ARRAY, n : INTEGER)
    DECLARE i, j : INTEGER
    DECLARE temp : INTEGER
    FOR i = 1 TO n - 1
        FOR j = 1 TO n - i
            IF arr[j] > arr[j + 1] THEN
                temp ← arr[j]
                arr[j] ← arr[j + 1]
                arr[j + 1] ← temp
            ENDIF
        ENDFOR
    ENDFOR
ENDPROCEDURE

— Compare adjacent elements, swap if out of order. Largest "bubbles" to end.
— Passes: n-1 passes needed.
— Best case: O(n) if already sorted (with optimisation).
— Worst/Average: O(n²).
— Not suitable for large datasets but easy to understand.

Activity (10 min): Trace bubble sort on [5, 2, 8, 1, 9]. Show array after each pass.

Plenary (5 min): "Why is bubble sort called 'bubble' sort?" Discuss: lighter (smaller) elements bubble up.

Homework: Implement bubble sort with early exit flag (stop if no swaps in a pass). Trace improved version.

Common Misconceptions:
• ❌ "Linear search is O(n) always" — Best case is O(1), worst is O(n)
• ❌ "Bubble sort is the best sorting algorithm" — Very slow for large data; use for education only

• Declaration: DECLARE Grid : ARRAY[1:3, 1:3] OF INTEGER
• Access: Grid[2, 3] → row 2, column 3.
• Initialisation with nested loops:
DECLARE Grid : ARRAY[1:3, 1:3] OF INTEGER
DECLARE r, c : INTEGER
FOR r = 1 TO 3
    FOR c = 1 TO 3
        Grid[r, c] ← 0
    ENDFOR
ENDFOR

• Row-major traversal: Outer loop = rows, inner = columns.
• Column-major traversal: Outer loop = columns, inner = rows.
• Applications: Game boards, matrices, timetables, image pixels, seating plans.
• Sum of row: FOR c = 1 TO 3: total ← total + Grid[r, c]
• Sum of column: FOR r = 1 TO 3: total ← total + Grid[r, c]

Weeks 1–9 Common Misconceptions Summary

1. Variables: Forgetting to declare before use. Using = instead of ← for assignment.
2. Selection: Missing ENDIF. Using = for comparison in pseudocode (CIE uses = for both, context determines).
3. Iteration: Infinite loops (forgetting to update counter). Off-by-one errors.
4. Subroutines: Confusing procedures and functions. Not understanding scope.
5. Recursion: Forgetting base case. Not understanding stack growth.
6. Arrays: Index out of bounds. Confusing 1-based vs 0-based indexing.
7. Records: Incorrect dot notation (Class[1].Name not Class.Name[1]).
8. Files: Not closing files. Reading past EOF.
9. Strings: Confusing 1-based indexing for substring. Forgetting LENGTH function.
10. General: Insufficient comments. Unmeaningful variable names. No input validation.

Address each explicitly with examples and practice.
