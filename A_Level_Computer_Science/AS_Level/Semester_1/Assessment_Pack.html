6. System Software Essay (Week 12)

System Software Essay Assessment

Time: 20 minutes  |  Marks: 9  |  CIE-style 9-mark essay

Question:

"Compare paging and segmentation as memory management techniques. In your answer, include:

  • how each technique works
  • advantages of each technique
  • disadvantages of each technique
  • a conclusion stating which is more suitable for a general-purpose operating system."

[9 marks]

Mark Scheme (Levels of Response)

LevelMarksDescriptor
48–9Detailed comparison; clear explanation of both; valid advantages/disadvantages; reasoned conclusion; well-structured
36–7Good explanation of both; some comparison; most advantages/disadvantages included; conclusion present
24–5Basic explanation of both; limited comparison; some advantages/disadvantages; weak conclusion
11–3Limited understanding; one technique described poorly; minimal comparison
00No relevant content

Indicative Content

Paging: Fixed-size pages; OS maintains page table; no external fragmentation; internal fragmentation possible; simple to implement; fast allocation.

Segmentation: Variable-size logical segments; segment table; no internal fragmentation; external fragmentation; matches program structure; protection per segment.

Conclusion: Paging generally preferred for general-purpose OS due to simplicity, no external fragmentation, easier management. Segmentation better for protection/sharing.

7. Security Case Study Test (Week 15)

Security Case Study Assessment

Time: 30 minutes  |  Marks: 25

Scenario:

Victoria Park Academy stores student records on a central file server. The server is accessible to all staff via the school network. Students use the network for coursework. Last week, the IT department discovered that several files had been encrypted and a ransom note was displayed on the server.

场景：维多利亚公园学院将学生记录存储在中央文件服务器上。所有教职员工可通过学校网络访问该服务器。学生使用网络完成课程作业。上周，IT部门发现多个文件被加密，服务器上显示勒索信息。

Q1. Identify three different types of malware that could have caused this incident. For each, briefly explain how it operates. [6 marks]

Q2. Explain how the malware might have entered the school's network. Include two different methods in your answer. [4 marks]

Q3. The school is considering its response. Evaluate the following options:

  (a) Pay the ransom [3 marks]

  (b) Restore from backups [3 marks]

  (c) Rebuild the server from scratch [3 marks]

Q4. Recommend technical, physical, and procedural security measures the school should implement to prevent future incidents. [6 marks]

Mark Scheme (Summary)

Q1: Ransomware (encrypts files/demands payment), Worm (self-replicates/spreads via network), Trojan (disguised as legitimate/creates backdoor). 2 marks each (type + explanation). | Q2: Phishing email (staff clicked link), infected USB (student plugged in), unpatched vulnerability, social engineering. Any 2 with explanation (4). | Q3a: May not get key/funds criminals/encourages attacks/no guarantee (3). | Q3b: Best option if backups exist/need recent backups/tested recovery/time to restore (3). | Q3c: Most secure/ensures clean system/loss of data if no backup/time consuming (3). | Q4: Technical (firewall, anti-malware, encryption, IDS), Physical (biometric access, CCTV, locked server room), Procedural (staff training, backup policy, incident response, password policy).
